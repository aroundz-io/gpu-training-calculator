주문 요약

내 데이터로 어떤 AI를 만들 수 있을까요?

코딩을 몰라도 괜찮습니다. 갖고 계신 데이터의 종류만 알면,
아래 예시에서 가장 비슷한 카드를 골라 그대로 시작하면 됩니다.

우리 회사 전용 상담 챗봇

이런 데이터가 있다면 — 고객센터 상담 기록, 자주 묻는 질문(FAQ), 카카오톡 상담 내보내기(CSV·엑셀·텍스트)

만드는 방법 — "질문 → 답변" 쌍으로 정리해 8B급 모델을 LoRA 파인튜닝. 상담 1,000~50,000건이면 충분합니다.

추천: Llama 3.1 8B · LoRA · 예시 200MB → 몇만 원대

사내 문서 지식 Q&A 봇

이런 데이터가 있다면 — 사내 매뉴얼, 규정집, 위키, 업무 문서 (PDF는 텍스트로 변환해서)

만드는 방법 — 문서를 "질문 → 문서 기반 답변" 형태로 가공해 LoRA 파인튜닝. 문서 원문은 사전학습식으로 추가 학습할 수도 있습니다.

추천: Qwen2.5 7B · LoRA · 예시 1GB → 십만 원대

내 문체로 글 쓰는 어시스턴트

이런 데이터가 있다면 — 내가 쓴 블로그·기사·소설·보고서 모음 (.txt, .md)

만드는 방법 — 원문 텍스트를 그대로 LoRA 학습하면 문체·어휘를 흡수합니다. 수 MB(책 몇 권 분량)로도 효과가 있습니다.

추천: Llama 3.1 8B · LoRA · 예시 50MB → 만 원 이하

상품 사진 자동 분류기

이런 데이터가 있다면 — 카테고리별로 정리된 상품·불량품·현장 사진 (폴더별 분류면 최적)

만드는 방법 — 클래스별 폴더에 담아 ZIP으로 올리고 이미지 분류(ViT) 학습. 클래스당 수백 장이면 시작할 수 있습니다.

추천: ViT-B/16 · 이미지 분류 · 예시 5,000장 → 만 원대

브랜드 스타일 이미지 생성

이런 데이터가 있다면 — 제품·브랜드·작품 사진 수십~수백 장 (각 이미지에 짧은 설명 텍스트가 있으면 더 좋음)

만드는 방법 — 확산 모델(SDXL 계열) 학습으로 우리 브랜드 느낌의 이미지를 생성하는 모델을 만듭니다.

추천: SDXL · 확산 모델 · 예시 300장 → 만 원대

전문 분야 특화 LLM

이런 데이터가 있다면 — 법률·의료·금융 등 전문 문서 수 GB~수십 GB (판례, 논문, 보고서)

만드는 방법 — 원문으로 전체 파인튜닝(계속 사전학습) 후, Q&A 데이터로 LoRA를 한 번 더 얹으면 전문가형 모델이 됩니다.

추천: Qwen2.5 72B · 전체 FT · 예시 20GB → 수백만 원대

학습 유형, 뭐가 다른가요?

유형비유언제 쓰나필요 데이터비용 감각
LoRA / QLoRA 파인튜닝가장 인기 · 입문 추천 기존 모델에 얇은 전문 지식 레이어를 덧입히기 말투·형식·업무 지식 등 특정 능력만 추가수백 건 ~ 수만 건수천 원 ~ 수십만 원
전체 파인튜닝 (Full FT) 모델 전체를 우리 데이터로 다시 다듬기 도메인 문서가 많고(수 GB↑) 깊은 특화가 필요할 때수 GB ~ 수십 GB수십만 ~ 수백만 원
사전학습 (Pre-training) 백지에서 모델을 새로 만들기 자체 언어모델이 필요한 기업·연구기관수백 GB ~ 수십 TB수천만 원 이상
이미지 분류 (ViT/CNN) 사진을 보고 종류를 맞히는 눈 만들기 상품 분류, 불량 검출, 문서 분류클래스당 수백 장~수천 원 ~ 수만 원
확산 모델 (SD 계열) 우리 스타일로 그림 그리는 손 만들기 브랜드 이미지 생성, 캐릭터·제품 렌더링수십 ~ 수천 장수천 원 ~ 수십만 원

자주 묻는 질문

Q. 코딩을 전혀 모르는데 가능한가요?
네. 엑셀로 질문/답변을 정리해 CSV로 저장할 수 있다면 충분합니다. 업로드하면 형식·언어·레코드 수를 자동 분석해 드립니다.

Q. 데이터가 너무 적어요 (수십 건).
수십 건으로는 학습보다 프롬프트(지시문)에 예시를 넣는 편이 낫습니다. 300건 이상 모이면 LoRA 효과가 나기 시작합니다.

Q. 개인정보가 섞여 있어요.
학습 전 반드시 마스킹하세요. 모델은 학습 데이터를 기억할 수 있어, 개인정보가 응답에 노출될 위험이 있습니다.

Q. PDF·한글(HWP) 문서는요?
텍스트(.txt/.md)로 변환 후 올려주세요. 표가 많은 문서는 CSV로 정리하는 것이 좋습니다.

Q. 데이터가 수십 GB가 넘어요.
별도 문의를 통한 FTP/SFTP 전송으로 처리해 드립니다.

GPU 실효 성능 학습 시간 단가 ($/GPU·시간) 총 비용 상대 비용
계산 방식 — LLM: 총 FLOPs ≈ 6 × 파라미터 × 토큰 (LoRA는 가중치 그래디언트 생략을 반영해 ≈ 0.67배, MoE 모델은 활성 파라미터 기준). 텍스트 용량 환산은 1토큰 ≈ 4바이트(평문 기준), 이미지는 장당 약 500KB로 가정합니다. 비전/확산/기상: 총 FLOPs ≈ 샘플당 순전파 FLOPs × 3(학습 배수) × 샘플 수 × 에포크. NVIDIA Earth-2 프리셋의 샘플당 연산량은 공개된 학습 비용(FCN3 ≈ 9만 GPU·시간 — H100 1,024개×78h 외, ERA5 1980–2016 / CorrDiff ≈ 5,000 A100 GPU·시간, CONUS)에 맞춰 보정한 추정치입니다. 학습 시간 = 총 FLOPs ÷ (GPU 수 × 이론 TFLOPS × MFU). FP8 미지원 GPU(A100 등)는 BF16으로 자동 대체되며, BF16이 없는 V100·T4는 FP16 텐서코어 수치를 사용합니다. DGX Spark는 구매형($3,999) 장비라 단가를 3년 감가상각+전기료 환산치(약 $0.2/시간)로 넣었고, 통합 메모리 대역폭(273GB/s)이 낮아 학습 시 MFU를 15~25%로 낮춰 보는 것이 현실적입니다.
주의 — 실제 시간은 데이터 파이프라인, 통신 병목, 시퀀스 길이, 배치 크기 등에 따라 달라질 수 있는 추정치입니다. 시간당 단가는 2025~2026년 주요 클라우드(Lambda, RunPod, CoreWeave 등) 온디맨드 시세 기준 기본값이며 직접 수정할 수 있습니다.

대용량 데이터 FTP 전송 안내

웹 업로드는 브라우저 특성상 5GB 이하를 권장합니다. 그 이상의 대용량 데이터(수십 GB~TB급, ZIP64 대형 압축 포함)는 아래 절차에 따라 전용 FTP/SFTP 전송으로 처리해 드립니다.

1. 별도 문의

bigdata@trainlab.io로 데이터 규모(용량·파일 수·형식)와 학습 목적을 보내주시면, 영업일 기준 1일 내 회신드립니다.

2. 전용 계정 발급

고객 전용 SFTP 계정(암호화 전송, 기간 한정)을 발급해 드립니다. 대용량 병렬 전송 도구(lftp, FileZilla, rclone 등) 사용법도 함께 안내됩니다.

3. 전송 및 자동 분류

전송이 완료되면 서버에서 자동으로 압축 해제·분류·검증(형식·언어 구성·레코드 수 분석)이 진행되고, 결과가 계정 대시보드에 연결됩니다.

4. 견적 확정 → 학습 시작

분석 결과 기준으로 견적이 확정되며, 결제 후 바로 학습 큐에 등록됩니다. 전송된 데이터도 동일한 보관 정책(학습 후 삭제 조치, 15일 후 영구 삭제 / 스토리지 서비스 이용 시 기간 내 안전 보관)이 적용됩니다.

※ 데모 사이트로, 실제 FTP 계정 발급·이메일 응대는 이뤄지지 않습니다.

가입 시 이용약관 및 데이터 보관 정책에 동의한 것으로 간주됩니다.
데모 모드 — 계정은 이 브라우저(localStorage)에만 저장됩니다.

TrainLab 이용약관 (발췌)

제3조 (서비스의 제공)

회사는 이용자가 업로드한 학습 데이터와 선택한 모델·GPU 구성에 따라 AI 모델 학습 서비스를 제공합니다. 요금은 사용한 GPU 시간을 기준으로 정산합니다.

제4조 (학습 데이터의 보관 및 삭제)

① 회사는 이용자가 업로드한 학습 데이터를 전송 및 저장 구간에서 암호화하여 안전하게 보관합니다.

② 학습 데이터는 학습 수행을 위한 목적으로만 임시 보관되며, 학습이 완료된 후에는 삭제 조치됩니다.

③ 임시 보관된 데이터는 학습 완료일로부터 15일이 경과한 후 영구 삭제함을 원칙으로 하며, 영구 삭제된 데이터는 어떠한 방법으로도 복구할 수 없습니다.

④ 이용자가 데이터를 계속 보관하고자 하는 경우 별도의 스토리지 서비스를 이용하여야 하며, 이 경우 보관 용량과 기간에 따른 보관료가 부과됩니다.

⑤ 스토리지 서비스 이용 시 데이터는 이용 기간 동안 안전하게 보관되며, 이용 기간이 만료되면 15일의 유예기간을 거쳐 영구 삭제됩니다. 유예기간 내 연장 결제 시 보관이 계속됩니다.

⑥ 회사는 이용자의 학습 데이터를 학습 수행 및 보관 목적 외의 용도(자체 모델 학습, 제3자 제공 등)로 사용하지 않습니다.

제5조 (환불)

학습 시작 후에는 사용한 GPU 시간만큼 비용이 정산되며, 미사용분은 환불됩니다. 스토리지 보관료는 개시 후 잔여 기간에 대해 일할 환불됩니다.

※ 본 사이트는 서비스 구조 시연용 데모로, 위 약관은 예시이며 실제 결제·데이터 저장·법적 효력이 없습니다.
